# 01_项目概览与范围

## 1.1 项目背景与核心愿景
EasyStay 酒店预订平台的设计初衷是为了构建一个高性能、可扩展且具备深度智能交互能力的端到端住宿服务解决方案。在当前数字化转型的大背景下，酒店行业对于预订系统的需求已不再局限于基础的库存管理和订单处理，而是涉及到了 LBS 地理位置服务、大数据驱动的城市经营统计以及大语言模型驱动的智能化顾问。本项目通过全栈技术的整合，旨在验证一种能够适配现代互联网环境的“三端互联”架构体系，即面向消费者的移动端应用、面向内部运营与商户的管理后台，以及作为核心支撑的弹性后端服务集群。

核心愿景在于通过技术手段消除信息差并提升用户决策效率。对于终端用户而言，EasyStay 不仅提供基础的筛选检索，更引入了基于地理位置的智能推荐和 AI 聊天助手，使用户能够通过自然语言获得精准的住宿建议。对于管理者和商户，平台则通过高度可视化的运营看板和严密的审核工作流，确保了资源发布的质量与业务数据的透明度。这种多维度的产品设计逻辑，决定了项目在技术实现上必须具备极高的健壮性和模块化程度。

## 1.2 平台全业务生命周期定义
本系统的业务闭环覆盖了从资源录入、审核上架、搜索发现、意图识别到最终下单支付与履约的全过程。在资源端，商户通过管理后台录入酒店元数据与房型库存，系统自动将其状态置为“待审核”，通过后台审核流转进入“已发布”状态后，方可在前端市场可见。在消费端，用户行为路径被设计为由定位服务作为锚点，首页通过城市统计数据引导用户进入特定市场的搜索列表。

搜索阶段是系统技术密度的集中体现。平台结合了传统的关系型数据库结构化查询与基于 AI 的自然语言意图解析。当用户输入模糊的需求描述时，AI 助手能够将其转化为后端的查询参数，从而实现从非结构化输入到结构化检索的平滑过渡。下单流程则通过严谨的状态机控制，确保了库存扣减、待支付锁位以及模拟支付后的状态同步逻辑。这种全链路的覆盖，使得 EasyStay 不仅仅是一个 Demo，而是一个具备完整业务逻辑的工程化原型。

## 1.3 核心技术栈深度解析
在后端架构层面，项目选型了基于 Node.js 生态的 Express 框架作为核心。选择 Express 是基于其轻量级中间件机制和对异步 I/O 的原生支持，能够完美适配酒店预订中常见的大量并发读请求。为了保证数据模型的一致性和开发效率，引入了 Sequelize 作为 ORM 工具。Sequelize 的关联查询能力极大简化了酒店（Hotel）、房型（RoomType）与订单（Order）之间复杂的跨表关系维护，并利用其钩子函数和事务支持，在关键操作如房型更新时保证了级联删除和数据完整性。

前端领域则采取了两套不同的演进路线。管理后台（admin-web）采用了 React 18 配合 Vite 构建体系，侧重于复杂的表单逻辑处理和数据可视化展示。Vite 的热重载和极速构建能力极大提升了开发体验，而 Ant Design 的引入则保证了 B 端界面的一致性和组件可靠性。移动端（mobile-app）则选用了 Taro 跨端框架，基于 React 语法体系实现了类似小程序的原生体验。Taro 的选型不仅是为了兼容 H5，更是为未来平滑迁移到微信小程序或其他移动端平台预留了空间。通过 NutUI 组件库，我们在移动端实现了细腻的交互动效和响应式布局。

## 1.4 智能化与地理信息服务集成
EasyStay 的技术亮点之一在于对第三方高性能服务的深度集成。智谱 GLM 大语言模型作为系统的大脑，被部署在后端 Service 层，负责处理聊天对话、意图解析以及酒店内容的风险预审。我们通过封装统一的 GLM 交互协议，使得系统具备了处理非结构化数据的能力，这在传统酒店系统中属于代际跨越的功能点。AI 不再是独立存在的模块，而是深度嵌入到了检索和审核的业务流程中。

与此同时，高德 Web 服务（AMAP）被集成用于解决 LBS 场景下的技术难题。从精准的 IP 定位到周边 POI 检索，每一个环节都经过了服务层的分级处理。系统不仅能根据用户当前位置推荐城市酒店，还能通过聚合查询展示每个城市的酒店分布密度。这种“AI + LBS”的双引擎驱动，赋予了 EasyStay 极强的市场竞争力，使其在满足基础预订需求的同时，能够提供差异化的智能服务体验。

## 1.5 部署架构与运维工程化
为了确保项目在演示环境和生产环境下的高度一致性，我们采用了容器化部署与自动化脚本相结合的运维方案。Docker Compose 被用于编排基础数据库设施，通过隔离的环境减少了版本冲突的可能性。Nginx 则作为流量分发的唯一入口，通过精细的子路径代理配置（/api、/admin、/mobile），实现了单机多端的分发能力，极大降低了服务器带宽 and 端口维护成本。

运维自动化通过 Shell 脚本进一步巩固。自研的 `update.sh` 发布脚本通过集成 Git 同步、依赖自动安装、PM2 进程热加载以及静态资源同步等指令，实现了一键式发布。这种工程化思维体现在项目的每一个细节中，从 .env 环境变量的读取到 GitHub Actions 的同步工作流，均旨在构建一个自动化程度高、人工介入低、具备自我复原能力的现代化 Web 系统。
